 Consuming Services

Climate Wizard datasets are exposed restfully using ArcGIS Server. You can consume the climate wizard data either as an ArcGIS Image Service or as an ArcGIS Map Service. In both these approaches you would need at a minimum an unique query string to be a part of the request (either as a part of the URL in case of MapServer or as a value to the mosaic name attribute in case of ImageServer). This query string directly corresponds to the option available on the Climate Wizard app, since there are numerous combinations possible, you can use this handy query reference documentation (PDF) that lists all the possible values that one could use to construct the query string either manually or programatically.

If you want to browse through the list of services and information of each of the service, here is a link to the ArcGIS Restful Services Directory.

Option 1. Image Services (Preferred approach)

ArcGIS Image service can be consumed using custom applications built using any of the ArcGIS Server APIs, such as the Climate Wizard app. This is the preferred approach and is the best (and fastest) way to access the climate wizard data. This approach also brings in a challenge of posting long URL's with lots of parameters and in most cases this would need a developer to use the API and send these parameters along.

Here is a sample request (URL encoded) http://www.climatewizard.org/ArcGIS/rest/services/TNC/US3Band/ImageServer/exportImage?size=668%2C495&format=png&transparent=true&noData=0&interpolation=RSP_NearestNeighbor&imageSR=102113&bboxSR=102113&bbox=-13934843.666294016%2C2146699.2774890885%2C-7399171.999800042%2C6989749.389636569&mosaicRule=%7B%22mosaicMethod%22%3A%22esriMosaicAttribute%22%2C%22where%22%3A%22Name%20%3D%20'c_ensemble_50_a2_tmean_14_2070_2099'%22%2C%22sortField%22%3A%22Name%22%7D&f=image The above long request is made up of two parts. Option 2. Map Service :

This custom application (Translator) allows you to access Climate Wizard Images service as a Map Service. This option could be used if Option 1 of using Image services is not suitable. The use cases for using this approach would be to use climate wizard services in clients which cannot consume or support Image services with custom attributes (Example: ArcGIS.com, ArcGIS on iPhone). You could also register these links in ArcGIS.com as items so that others could search and add these services directly from ArcGIS.com and build great mashups.

Think of this as a map url shortner which helps to tranlate between a short map service url into a rather complex and long image service url. This is being used in the Climate Wizard app in the "View in ArcGIS.com" link.

Sample Request (same map as above) http://www.climatewizard.org/translator/US3Band/c_ensemble_50_a2_tmean_14_2070_2099/MapServer To use this just copy the map server link listed in the section "Current Climate Map Information" or to learn more use this link Translator
